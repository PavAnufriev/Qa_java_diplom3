# QA Java Диплом 3 - Тестирование Stellar Burgers

Автотесты для сайта [Stellar Burgers](https://stellarburgers.nomoreparties.site/) с использованием Selenium WebDriver, Page Object паттерна и Allure отчетов.

## Что тестируется

### Регистрация
- ✅ Успешная регистрация пользователя
- ✅ Ошибка для некорректного пароля (менее 6 символов)

### Вход в систему
- ✅ Вход по кнопке «Войти в аккаунт» на главной странице
- ✅ Вход через кнопку «Личный кабинет»
- ✅ Вход через кнопку в форме регистрации
- ✅ Вход через кнопку в форме восстановления пароля

### Раздел «Конструктор»
- ✅ Переход к разделу «Булки»
- ✅ Переход к разделу «Соусы»
- ✅ Переход к разделу «Начинки»

## Технологии

- **Java 11** - язык программирования
- **Maven** - сборка проекта
- **JUnit 4** - фреймворк для тестирования
- **Selenium WebDriver** - автоматизация браузера
- **WebDriverManager** - автоматическое управление драйверами
- **Allure** - генерация отчетов
- **Page Object** - паттерн проектирования для тестов

## Поддерживаемые браузеры

- **Google Chrome** (по умолчанию)
- **Яндекс Браузер**

## Запуск тестов

### Запуск всех тестов в Chrome
```bash
mvn clean test
```

### Запуск тестов в Яндекс Браузере
```bash
mvn clean test -Dbrowser=yandex
```

### Запуск конкретного класса тестов
```bash
mvn clean test -Dtest=RegistrationTest
```

### Запуск конкретного теста
```bash
mvn clean test -Dtest=RegistrationTest#testSuccessfulRegistration
```

## Генерация Allure отчета

### Генерация отчета
```bash
mvn allure:report
```

### Открытие отчета
```bash
mvn allure:serve
```

## Настройка для Яндекс Браузера

Для работы с Яндекс Браузером укажите пути в системных свойствах:

```bash
mvn clean test -Dbrowser=yandex 
  -Dyandex.binary.path="C:\Users\%USERNAME%\AppData\Local\Yandex\YandexBrowser\Application\browser.exe"
  -Dyandex.driver.path="C:\path\to\chromedriver.exe"
```

## Особенности реализации

### Page Object паттерн
Каждая страница представлена отдельным классом с методами для взаимодействия с элементами:
- `MainPage` - главная страница
- `LoginPage` - страница входа
- `RegisterPage` - страница регистрации
- `ForgotPasswordPage` - страница восстановления пароля

### Базовый класс тестов
`BaseTest` содержит общую логику:
- Настройка WebDriver
- Управление браузерами
- Cleanup после тестов

### Генерация тестовых данных
`TestDataGenerator` создает уникальные тестовые данные для каждого запуска.

### Allure аннотации
- `@Epic` - группировка по функциональности
- `@Feature` - группировка по фичам
- `@Step` - детализация шагов теста
- `@Description` - описание теста
- `@Severity` - важность теста

## Результаты тестирования

После запуска тестов:
1. Результаты сохраняются в `target/allure-results/`
2. Генерируется Allure отчет с детальной информацией
3. Отчет включает скриншоты, логи и временные метки

## Устранение неполадок

### Проблемы с WebDriver
- Убедитесь, что браузер установлен
- Проверьте пути к исполняемым файлам
- WebDriverManager автоматически загружает драйверы

### Проблемы с тестами
- Проверьте доступность сайта
- Убедитесь в стабильности интернет-соединения
- Элементы страницы могут изменяться - обновите локаторы
